# 🚨 緊急修復：更新居家子分類欄位

## 問題原因

Supabase RLS (Row Level Security) 阻止了後端 API 使用 `anon` key 更新資料。

## 解決方案：直接在 Supabase 執行 SQL

### 步驟 1：登入 Supabase Dashboard

1. 前往 https://supabase.com/dashboard
2. 選擇您的專案：`uuhxyaggbrhaytmyaqmo`

### 步驟 2：打開 SQL Editor

1. 左側選單點擊 **SQL Editor**
2. 點擊 **New query** 創建新查詢

### 步驟 3：複製並執行 SQL

1. 打開檔案：`d:\AI建站\ai-matching\docs\UPDATE-HOME-FORM-CONFIG.sql`
2. **全選複製** 整個檔案內容
3. **貼上** 到 Supabase SQL Editor
4. 點擊 **RUN** 執行

### 步驟 4：驗證結果

執行完成後，最下方會顯示驗證結果表格，確認所有子分類都顯示：
- ✅ 3 個欄位（表示成功）

如果看到：
- ❌ 空物件 {} → 表示更新失敗，請再次執行
- ❌ NULL → 表示更新失敗

### 步驟 5：測試首頁

1. 清除瀏覽器快取（Ctrl+F5）
2. 訪問 http://localhost:3000/index.html
3. 選擇「居家」分類
4. 選擇「清潔服務」子分類
5. **應該會看到 3 個動態欄位：施作坪數、清潔類型、樓層與電梯**
