# 還原點：種子廠商實作前

**用途**：記錄「種子廠商模式」實作前的系統狀態，日後若要還原或比對時可依此為基準。  
**建立時間**：種子廠商功能首次實作前。  
**相關評估**：`docs/封測白手套入駐與權限-評估.md` 第〇節（種子廠商模式）。

---

## 1. 資料庫：廠商相關表結構（實作前）

### 1.1 `public.manufacturers`

- **定義檔**：`docs/custom-products-schema.sql`（主表）、`docs/add-manufacturers-user-id.sql`（user_id）、`docs/add-manufacturer-logo.sql`（logo_url）。
- **欄位（實作前）**：
  - `id` UUID PK
  - `name` TEXT NOT NULL
  - `description` TEXT
  - `categories` TEXT[] DEFAULT '{}'
  - `contact_json` JSONB DEFAULT '{}'
  - `location` TEXT, `address` TEXT
  - `capabilities` TEXT[] DEFAULT '{}', `certifications` TEXT[] DEFAULT '{}'
  - `rating` DECIMAL(3,2), `review_count` INTEGER
  - `min_order_quantity` INTEGER, `lead_time_days` INTEGER
  - `is_active` BOOLEAN DEFAULT TRUE
  - `verified` BOOLEAN DEFAULT FALSE
  - `created_at`, `updated_at` TIMESTAMPTZ
  - `user_id` UUID REFERENCES auth.users(id)（見 add-manufacturers-user-id.sql）
  - `logo_url` TEXT（見 add-manufacturer-logo.sql）
- **當時沒有**：`account_status`、`expires_at`、`claim_token`、`vendor_source`（或任何「種子／來源」標記）。

### 1.2 `public.manufacturer_portfolio`

- **定義檔**：`docs/manufacturer-portfolio-schema.sql` 及後續 add-*.sql（如 category_type、series 等）。
- 與種子廠商相關：廠商作品由廠商本人透過 API 上傳，或後台／SQL 寫入；無「僅種子可編輯」等區分。

### 1.3 其他相關表

- `vendor_assets`（廠商素材庫）、`media_collections`（含 manufacturer_id）、`manufacturer_collections` 等：皆依 `manufacturers.id` 或 `user_id` 關聯，**無**依「種子／非種子」的欄位或政策。

---

## 2. 後端 API（廠商相關，實作前）

| 方法與路徑 | 說明 | 權限 |
|------------|------|------|
| GET /api/me/manufacturer | 取得當前登入者的廠商資料 | 登入且已建廠商 |
| POST /api/me/manufacturer | 建立「我的廠商資料」（名稱必填，綁定當前 user_id） | 登入，且該 user 尚無廠商 |
| PATCH /api/me/manufacturer | 更新廠商資料（名稱、描述、地點、contact_json 等） | 登入且為該廠商之 user_id |
| GET /api/manufacturers/:id | 單一廠商詳情（對外） | 公開或依 is_active |
| POST /api/manufacturers/:id/portfolio | 上傳廠商作品（系列圖／對照圖） | 登入且為該廠商之 user_id |
| （其他） | 廠商作品 CRUD、vendor_assets、collections 等 | 同上，依 user_id 與 manufacturer 歸屬 |

- **當時沒有**：管理員「代建廠商」或「指定 user_id 建立廠商」的 API；種子廠商若由平台代建，需透過 **Supabase 後台／SQL** 直接 INSERT `manufacturers` 並設定 `user_id`。

---

## 3. 曝光與權限邏輯（實作前）

- **首頁／搜尋／廠商列表**：以 `is_active = true` 篩選廠商；**未**依 `account_status`、`vendor_source` 或到期日過濾。
- **廠商端權限**：依「是否登入 + 是否為該廠商之 user_id」判斷可否編輯、上傳作品；**沒有**依方案等級或「種子／非種子」的差別權限。
- **設計端／訪客**：可瀏覽 is_active 廠商與作品；無變更。

---

## 4. 若日後要還原或比對

- **還原**：若有新增欄位（例如 `vendor_source`），可選擇不刪除欄位、僅停止使用；若有新增 API（例如 POST /api/admin/seed-manufacturer），可停用或移除該路由；既有 API 與既有欄位語意不變即為還原。
- **比對**：以此文件為「實作前」基準，與當前 schema、API 列表、權限說明對照即可。

---

## 5. 實作後新增項目

- **新增欄位**：`public.manufacturers.vendor_source` (TEXT)、`public.manufacturers.expires_at` (TIMESTAMPTZ)。見 `docs/add-manufacturers-vendor-source.sql`、`docs/add-manufacturers-expires-at.sql`。
- **新增 API**：`POST /api/admin/seed-manufacturer`（僅管理員）。建立時寫入 `expires_at = now() + 90 天`。
- **權限與曝光**：種子廠商（`vendor_source = 'seed'`）不得編輯廠商資料、不得新增／編輯／刪除作品（PATCH /api/me/manufacturer、POST/PUT/DELETE portfolio 回傳 403）。管理員可代為上傳（同一 POST portfolio API）。所有「公開列出廠商」的查詢皆過濾 `expires_at`（NULL 或 > now()）；單一廠商詳情若已過期回 404。
- **新增文件**：`docs/種子廠商入駐操作手冊.md`、`docs/add-manufacturers-vendor-source.sql`、`docs/add-manufacturers-expires-at.sql`。
