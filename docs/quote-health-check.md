# 報價健檢（Quote Health Check）

## 目的

- 讓專家知道自己的報價在「同分類、同計價單位」的市場中落在哪個區間（前段／中段／後段）。
- **不建議降價、不與同業競價**；建議以**上傳作品、提供高品質服務證明**爭取客戶。

## 邏輯

- **比較範圍**：同一 `category` + 同一 `unit` 的其他專家有效報價（`status = 'active'`，排除自己）。
- **代表價格**：單筆報價以 `(price_min + price_max) / 2` 作為比較用單價。
- **分布**：對同業報價排序後取 P25、P50、P75。
- **區間**：
  - 我的單價 **> P75** → **前段**（高於約 75% 同業）
  - 我的單價 **< P25** → **後段**
  - 其餘 → **中段**
- **最少樣本**：同分類同單位至少 3 筆同業報價才顯示區間，否則僅顯示「尚無足夠資料可比較」+ 建議以作品證明價值。

## 文案（不導向降價）

| 區間 | 主文 | 建議 |
|------|------|------|
| 前段 | 您的報價落在同分類報價的 **前段**（高於約 75% 同業）。 | 可透過 **作品集與案例** 說明服務差異與品質，吸引重視成果的客戶。 |
| 中段 | 您的報價落在同分類報價的 **中段**，與多數同業區間相近。 | 可透過 **完整作品與經歷** 讓客戶更容易辨識您的優勢。 |
| 後段 | 您的報價落在同分類報價的 **後段**。 | 建立口碑與案例後，可適度在報價中反映您的價值；**上傳作品** 有助客戶理解您的服務水準。 |

## API

- **GET /api/expert/quote-health-check**
- 需登入（Bearer token）。
- 回傳：`{ items: [...], message }`
  - 每個 `item`：`listing_id`, `title`, `category`, `unit`, `my_price`, `band`（前段/中段/後段）, `message`, `suggestion`, `peer_count`, 可選 `p25`, `p50`, `p75`。

## 前端

- **我的報價**（`expert/my-listings.html`）：上方有「報價健檢」按鈕，點擊後呼叫 API 並在頁內顯示結果。
- **媒合專案 / 可媒合專案**：報價區塊內提供「報價健檢」連結，導向我的報價頁。
