# MatchDO 合做 — 使用者操作教學規劃

本文件為**教學規劃**，尚未撰寫實際教學內容。先界定對象、介面範圍與章節結構，後續可依此產出圖文／影片教學。

---

## 一、教學對象與目標

| 對象 | 目標 |
|------|------|
| **設計者／想訂製產品的人** | 會用「產品設計」生成草圖、選參考圖（含廠商素材庫）、看懂步驟「描述/選圖 → 生成 → 選廠商/下單」，並能到圖庫或廠商頁找廠商。 |
| **廠商／接單方** | 會用廠商控制台、上傳作品、**素材管理**（上傳可給設計者當參考的造型/材質圖）、看詢價與設定廠商資料。 |
| **一般訪客** | 能瀏覽首頁靈感牆、了解本站是「設計 + 找廠商」平台，知道登入後可設計與找廠商。 |

## 二、介面與功能對照（教學涵蓋範圍）

依目前介面整理，教學需涵蓋的區塊如下。

### 2.1 全站共通

- **頂部導覽**
  - 首頁、服務媒合、訂製品（客戶／供應商）、方案與定價、登入／帳號。
  - 「我的功能」下拉：**設計／找廠商**（客製產品首頁、建立新產品、我的數位資產、圖庫找廠商）、**開店／接單**（廠商控制台、我的廠商作品、素材管理、訂製詢價列表、廠商資料設定等）。
- **語系**：中文／英文（若有）。
- **登入／註冊**：從哪裡進入、註冊後可做什麼。

### 2.2 首頁（靈感牆）

- 三種版面或篩選：用戶設計、廠商對比、作品集（若有）。
- 分類篩選、單一作品點進去可看到什麼（大圖、廠商詳情連結等）。
- **不教**：後台或進階設定；首頁靈感牆版面依規格不變動，僅說明「這是什麼」。

### 2.3 設計／找廠商

- **客製產品首頁**：入口說明，引導到「建立新產品」或「我的數位資產」。
- **產品設計頁**（你截圖的畫面）
  - **步驟條**：「描述/選圖 → 生成 → 選廠商/下單」— 明確寫出三步驟，讓使用者知道現在在第幾步、接下來要做什麼。
  - **描述**：描述你想生成的產品（風格、材質、結構、顏色）；可從參考圖的「從此圖產生描述」貼過來。
  - **分類**：主分類（必選）、子分類（必選），會影響生成類型。
  - **參考圖**：
    - 本機上傳：點「+」上傳自己的圖。
    - **從廠商素材庫選擇**：先選好主分類後按鈕，會依分類列出廠商素材，點選即加入參考圖；卡片上會顯示廠商名稱與連結，方便之後找該廠商。
  - **進階 Seed**：選填，給進階使用者。
  - **生成圖片**：點數說明（無參考圖 15 點／有參考圖 20 點）、按鈕操作與生成後畫面。
  - **生成後**：結果預覽、儲存到數位資產、建議下一步（我的數位資產、圖庫找廠商）。
- **實境模擬**：環境／人物圖 + 產品圖，套用至實境；點數約 20 點；結果不下載則不存數位資產。
- **圖樣提取**：僅能從數位資產選圖；解析度與下載說明。
- **我的數位資產**：已生成的圖列表、再設計／實境模擬／圖樣提取入口、展示在首頁與否（若開放）。
- **圖庫找廠商**：瀏覽廠商或作品、點進廠商詳情或詢價。

### 2.4 開店／接單（廠商端）

- **廠商控制台**：步驟說明「上傳作品／設定分類 → 收到詢價 → 設計討論 → 報價／接單」；統計與快速入口。
- **我的廠商作品**：上傳作品（系列圖／對照圖）、分類、作品名稱、亮點、標籤、是否公開等。
- **素材管理**（廠商素材庫）
  - 用途：上傳「可給設計者當參考圖」的造型／結構／材質圖。
  - 操作：主分類（必選）、子分類（選填）、標題、說明、用途、上傳圖片。
  - 設計端會依**設計當下選的分類**載入這些素材，並顯示廠商名稱與連結。
- **訂製詢價列表**：查看需求、回覆或報價（若有）。
- **廠商資料設定**：名稱、介紹、聯絡方式、服務地區、LOGO、廠商頁預覽連結。

### 2.5 其他

- **方案與定價**：點數、方案（若對外開放）。
- **找製作方／訂製需求**：若有獨立頁面，簡短說明入口與用途。

## 三、建議教學章節結構（大綱）

以下為建議的「教學文件／影片」章節，可依對象拆成「設計者篇」「廠商篇」或一本通。

1. **開始使用 MatchDO**
   - 首頁在做什麼、誰適合用、登入／註冊入口。
2. **用產品設計生成草圖（設計者）**
   - 步驟：描述 → 選分類 → 選參考圖（本機或廠商素材庫）→ 生成。
   - 參考圖區塊：本機上傳 vs 「從廠商素材庫選擇」、為何要先選分類。
   - 點數說明、生成後儲存與下一步（數位資產、圖庫找廠商）。
3. **實境模擬與圖樣提取**
   - 何時用、圖片從哪裡選、點數與結果存放方式。
4. **我的數位資產與圖庫找廠商**
   - 數位資產裡能做什麼（再設計、當實境/圖樣來源）；圖庫找廠商怎麼用、如何連到廠商頁。
5. **廠商端：上傳作品與設定**
   - 廠商控制台步驟說明；我的廠商作品上傳與分類。
6. **廠商端：素材管理**
   - 為何要上傳素材、設計者會怎麼看到、主分類／子分類與標題怎麼填、列表與編輯／刪除。
7. **詢價與接單流程**
   - 收到詢價、設計討論、報價／接單（若已實作）。
8. **常見問題**
   - 點數、分類選錯、找不到廠商素材、廠商頁連結等。

## 五、教學網頁及截圖適合放置的位置

以下建議**前台使用者**操作教學的網頁與截圖要放在哪裡，與後台 `/admin/` 的「知識庫」區分開。**截圖是流程示意用，嵌在教學頁內，不另做獨立頁。**

### 5.1 教學網頁放在「頁面」的哪裡（站內入口）

**頂部選單已很滿，不再加新主選單項目。** 使用說明改以**頁尾＋情境入口**（必要時可收進既有下拉）即可。

| 位置 | 建議做法 |
|------|----------|
| **頂部導覽列** | **不新增**「使用說明」主選單，避免選單過長。若未來一定要從頂部進，可改為收進既有下拉，例如「**我的功能**」下拉裡加一項「使用說明」→ `/help/`（與設計／開店並列），或「**客製產品**」下拉最下方加「使用說明」。 |
| **頁尾（Footer）** | **主要入口**：**連結**區在「首頁」「產品設計」「訂製產品」「聯絡我們」旁加 **「使用說明」** → `/help/`。**footer-menu** 區把目前連到 `#` 的「Help」「FAQs」改為「Help」→ `/help/`、「FAQs」→ `/help/faq.html`。 |
| **情境入口** | 產品設計頁、廠商控制台等可加一小段「第一次使用？<a href="/help/product-design.html">看教學</a>」，連到對應單元，不佔頂部空間。 |

實作時：**不**在 `site-header.js` 頂部主選單加新連結；頁尾在 `public/partials/footer.html` 的「連結」「footer-menu」加上或改寫為上述連結；情境入口在對應頁面加一小行即可。

### 5.1.1 如何讓「不會用」的使用者馬上看到教學

對象是**剛進站的人**——可能只看到首頁（靈感牆），還沒點進客製產品或控制台，不一定知道怎麼用。所以要讓教學在**進站第一眼就能看到**，而不是先進某個頁面才發現。

| 方式 | 做法 | 對象／時機 |
|------|------|------------|
| **首頁（進站第一眼）** | 在**首頁**（`/`、`/index.html`）——多數人進站的第一個畫面——放一個**明顯但不搶主視覺**的入口，例如：靈感牆上方或標題旁一行「第一次來？<a href="/help/">了解怎麼設計與找廠商</a>」，或一小塊可關閉的橫幅／卡片「不知道從哪開始？→ <a href="/help/">使用說明</a>」。**不要求**使用者先點進客製產品或控制台，一進站就有機會看到。 | **剛進站、還不一定會用的人**。 |
| **客製／產品設計／廠商控制台** | 在客製產品首頁、產品設計頁、廠商控制台再放對應單元的教學連結，給**已經點進去**的人。 | 已決定要設計或接單的人。 |
| **固定小按鈕（可選）** | 全站**右下角**「？」或「使用說明」小按鈕，任何頁（含首頁）都能點進 `/help/`。 | 剛進站或逛到任何頁都能找到。 |
| **首次造訪輕提示（可選）** | 偵測第一次造訪（localStorage），在**首頁**顯示一次小橫幅或輕 modal「不知道從哪開始？<a href="/help/">看使用說明</a>」，可關閉＋「不再顯示」。 | 新訪客第一眼就收到引導。 |
| **我的功能下拉 ＋ 頁尾** | 登入後「我的功能」加「使用說明」；頁尾「連結」與 footer-menu 有使用說明／Help／FAQs。 | 補足入口。 |

**建議優先做**：**首頁**就露出「第一次來？了解怎麼設計與找廠商」或「使用說明」連結（橫幅／標題旁／固定小按鈕擇一），讓剛進站的人**不必先進控制台或客製頁**就能看到教學。其餘入口（客製頁、控制台、頁尾、我的功能）再補。

**首頁「寫在哪」具體位置**（實作時可直接加）：

- **檔案**：`public/iStudio-1.0.0/index.html`（首頁由 server.js 的 `/`、`/index.html` 指向此檔）
- **插入位置**：在 `#media-wall-section` 內的 `.media-wall-inner` **最上方**，也就是在「搜尋＋篩選列」（搜尋框、全部／設計圖／對照圖／系列圖）**之前**。
- **對應程式碼**：約第 117 行 `<div class="container-fluid media-wall-inner py-3">` 的下一行（目前下一行是 `<style>`），可在 `</style>` 之後、第 209 行 `<div class="d-flex flex-wrap align-items-center justify-content-center gap-2 mb-2">`（搜尋列）**之前**，新增一塊區塊，例如：
  ```html
  <div class="text-center py-2 mb-2" id="home-help-cta">
    <span class="text-muted small">第一次來？</span>
    <a href="/help/">了解怎麼設計與找廠商</a>
  </div>
  ```
  或可關閉的小橫幅／卡片（含「不再顯示」需配合 localStorage）。這樣一進站、捲動前就能看到教學入口。

### 5.2 建議 URL 與資料夾

| 用途 | 建議路徑 | 說明 |
|------|----------|------|
| **教學首頁（目錄）** | `/help/` | 使用者操作教學入口，列各章連結。 |
| **實體資料夾** | `public/help/` | 與 `public/client/`、`public/custom/` 同層。 |
| **各單元頁** | `/help/product-design.html`、`/help/vendor-materials.html` 等 | 依「三、章節結構」一單元一頁。 |
| **截圖** | `public/help/images/` | **僅供流程示意**，放在此資料夾，**嵌在教學頁內**（如 `<img src="/help/images/xxx.png">`），不為每張截圖做獨立頁。 |

### 5.3 教學頁結構與截圖角色

- **一單元一頁**：例如「產品設計」「廠商素材管理」各一頁 HTML，頁內用文字＋**內嵌截圖**說明步驟。
- **截圖**：流程示意用（例如：步驟條長怎樣、從廠商素材庫選擇的畫面、素材管理列表）。截圖檔放在 `public/help/images/`，在對應教學頁用 `<img>` 或圖＋說明並排呈現，**不**為每張圖另開一頁。
- 範例結構：

```
public/help/
  index.html              # 教學目錄（章節列表）
  product-design.html     # 產品設計：文字＋內嵌截圖（如 步驟條、參考圖區、廠商素材庫 modal）
  vendor-materials.html   # 廠商素材管理：文字＋內嵌截圖（上傳表單、列表）
  faq.html
  images/                 # 流程示意截圖，僅供上述頁面引用
    product-design-steps.png
    vendor-assets-modal.png
    ...
```

### 5.4 截圖注意事項

- **檔名**：語意化（如 `product-design-steps.png`），方便在教學頁引用與日後替換。
- **語系**：**建議中英文都截圖**。站內使用說明與介面支援中/英切換，若教學圖與當前語系一致，使用者較易對照。做法可二擇一或並行：
  - **分資料夾**：`public/help/images/zh/`、`public/help/images/en/`，同一場景各存一份（如 `zh/product-design-steps.png`、`en/product-design-steps.png`），教學頁依語系載入對應路徑。
  - **檔名區分**：如 `product-design-steps-zh.png`、`product-design-steps-en.png`，頁面用語系切換 `src`。
- **更新**：介面改版時記得更新對應截圖；若中英文都截，兩邊都要更新。

### 5.5 對應使用說明頁（/help/）各段落的截圖建議

目前 `/help/` 上的介紹若要補**教學圖說**（圖＋說明），可依下表決定要截哪些畫面。截完後放到 `public/help/images/`，再在說明頁或獨立教學單元中貼上並加圖說即可。

| 說明段落 | 建議截的畫面 | 用途 |
|----------|----------------|------|
| **設計者／想訂製的人**：產品設計、描述、分類、參考圖、廠商素材庫 | 1. **產品設計頁**整頁或上半：步驟條（描述/選圖 → 生成 → 選廠商）、描述框、主分類／子分類、參考圖區（含「從廠商素材庫選擇」按鈕）。<br>2. 點「從廠商素材庫選擇」後的 **modal**：依分類列出的廠商素材卡片（有廠商名、連結）。 | 讓讀者對應「輸入描述、選分類與參考圖」「可從廠商素材庫選參考圖」與「前往產品設計」的實際畫面。 |
| **廠商／接單方**：廠商控制台、上傳作品、素材管理、詢價 | 1. **廠商控制台**首頁（統計／步驟說明、快速入口）。<br>2. **素材管理**頁：上傳表單（主分類必選、標題、圖片）與素材列表。 | 對應「上傳作品、素材管理、查看詢價」與「前往廠商控制台」。 |
| **常見問題：點數怎麼算？** | **方案與定價**頁（點數與方案說明）、或 **產品設計頁**上生成按鈕旁的點數提示（無參考 15／有參考 20）。可再加 **我的點數**頁。 | 對應「可至方案與定價與我的點數查看」。 |
| **常見問題：從廠商素材庫選擇時沒有圖？** | **產品設計頁**：已選主分類後，打開「從廠商素材庫選擇」modal 的畫面——可截兩種：(1) 有素材時列表；(2) 無素材時「此分類下尚無廠商素材」提示。 | 對應「請先選好主分類」「依分類載入」「尚無廠商上傳會顯示…」。 |
| **常見問題：廠商如何讓設計者看到我的素材？** | **廠商控制台 → 素材管理**：上傳表單（主分類必選、標題、上傳圖片）與成功列出素材的畫面。 | 對應「素材管理、主分類必選、設計者選相同分類時可看到」。 |

**結論**：要製作目前介紹的教學圖說，**至少需要節**以下幾類圖即可覆蓋全文（**中英文介面各截一份**，見 5.4 語系說明）：

1. **產品設計頁**：步驟條＋描述＋分類＋參考圖區（含「從廠商素材庫選擇」）。
2. **從廠商素材庫選擇 modal**：有素材列表／無素材提示（各一）。
3. **廠商控制台**首頁。
4. **廠商素材管理**：上傳表單＋素材列表。
5. **方案與定價**或產品設計上的點數說明；**我的點數**頁（可選）。

其餘（實境模擬、圖樣提取、數位資產等）若日後在說明頁加段落，再依同方式補對應截圖即可；語系同樣建議中英文都截。
