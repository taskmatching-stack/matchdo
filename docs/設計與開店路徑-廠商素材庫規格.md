## 一、目的與範圍

| 項目 | 說明 |
|------|------|
| **路徑明確化** | 讓「設計」與「開店」的**操作流程**更清晰易懂（導覽語意、步驟提示、完成後下一步建議）；**不更動首頁靈感牆**的版面與功能。 |
| **廠商素材庫** | 廠商可依分類上傳「素材」（造型、結構、材質樣板等），設計者在設計時可選用這些素材當參考圖，形成「廠商素材庫」；類似現有數位資產庫的選圖模式，但來源為廠商提供，利於設計符合該廠能力的產品（例如各廠手機殼造型不同）。 |

### 重要約束：首頁靈感牆不變動

- **首頁靈感牆**（媒體牆／設計圖／對照圖／系列圖等）的**版面、篩選、卡片、lightbox、URL 行為**均維持現狀，不做改動。
- 路徑明確化**僅限**：導覽選單的語意區分、產品設計頁／廠商端等**各功能頁內**的步驟提示與「建議下一步」，讓使用者知道「現在在做什麼、接下來該做什麼」，不涉及首頁內容改版。

本文件僅做規格與流程規劃，不涉及具體程式實作。

## 二、設計 vs 開店：操作流程更清晰（首頁靈感牆不變）

### 2.1 兩條主線定義

| 主線 | 角色 | 主要目標 | 典型入口與頁面 |
|------|------|----------|----------------|
| **設計** | 設計者／發案方 | 產出設計圖、找廠商、下單／詢價 | 產品設計、數位資產、圖庫找廠商、實境模擬、圖樣提取、方案（設計／生圖用） |
| **開店** | 廠商／接單方 | 展示作品、收詢價、報價、接單營運 | 廠商控制台、我的廠商作品、詢價列表、方案（開店／方案用） |

**約定**：以下建議**僅針對導覽與各功能頁內的操作流程**，**不更動首頁靈感牆**的版面、篩選、卡片或互動。

### 2.2 操作流程清晰化建議

#### 2.2.1 導覽選單語意區分

- **做法**：在全站導覽（navbar／選單）中，讓「設計相關」與「廠商／開店相關」的項目在**文案或分組**上更清楚。
  - 設計向：產品設計、我的數位資產、圖庫找廠商等，語彙一致用「設計／找廠商」。
  - 開店向：廠商控制台、我的廠商作品、詢價列表等，語彙一致用「開店／接單」。
- **不變動**：首頁本身仍是靈感牆；僅透過選單讓「從首頁要去做設計」或「要去做開店」的下一步更直覺。

#### 2.2.2 產品設計頁：步驟提示

- **做法**：在**產品設計頁**（custom-product 等）以簡短步驟條標示流程，例如「描述／選圖 → 生成 → 選廠商／下單」。
- **效益**：使用者清楚「現在在哪一步、下一步該做什麼」，無需改首頁。

#### 2.2.3 廠商端：步驟提示

- **做法**：在**廠商控制台／作品管理頁**以步驟條標示開店流程，例如「上傳作品／設定分類 → 收到詢價 → 設計討論 → 報價／接單」。
- **效益**：廠商知道接單流程的順序與建議下一步。

#### 2.2.4 完成一步後提示下一步

- **設計端**：例如生成完草圖後，在**該頁**明顯提示「加入數位資產」或「去找廠商」；從圖庫選完圖後提示「可帶入產品設計當參考圖」。
- **廠商端**：例如上傳作品後，在**該頁**提示「此作品將在圖庫被搜到」或「到詢價列表回覆客戶」。
- **效益**：每完成一個關鍵步驟就有明確的「建議下一步」，操作路徑更連貫；皆在**功能頁內**完成，不涉及首頁靈感牆。

#### 2.2.5 可選：登入後情境引導

- **做法**：登入後（或從選單進入「我的」時）可選擇「這次要**設計產品**還是**管理廠商／接單**」，導向對應起點頁。
- **說明**：僅為導向與情境選擇，不改變首頁靈感牆內容或結構。

### 2.3 小結（操作流程清晰化）

- **首頁靈感牆**：維持現狀，不變動。
- **可調整處**：導覽選單語意與分組、**產品設計頁／廠商端等各功能頁內**的步驟條與「建議下一步」提示。
- 實作時可**漸進**：先做選單語意與步驟提示，再視需要補情境引導。

## 三、廠商素材庫：規格詳述

**目前狀態**：本節為規格與資料表設計，**尚未實作**。實作時需建立廠商素材資料表（見 §3.2）與 API，再依實作順序（§3.6）開發廠商端上傳與設計端選圖。

### 3.1 概念與定位

| 對比項 | 數位資產庫（現有） | 廠商素材庫（本規格） |
|--------|--------------------|------------------------|
| **擁有者** | 設計者本人 | 各廠商 |
| **內容** | 設計者生成／上傳的圖 | 廠商上傳的造型、結構、材質等素材圖 |
| **用途** | 設計時當參考圖、實境模擬／圖樣提取選圖 | 設計時當參考圖，表示「依該廠造型／結構設計」 |
| **版權／授權** | 設計者自有 | 廠商上傳即視同授權站上設計者在設計流程中使用 |
| **典型情境** | 用自己的草圖再改、圖樣提取 | 各廠手機殼造型不同，選 A 廠造型當參考再設計，可執行度更高 |

目標：設計者在「產品設計」的參考圖來源，除「本機上傳」與「我的數位資產」外，增加**「廠商素材庫」**；選用方式與數位資產庫類似（依分類／廠商篩選後選圖帶入），但資料來源與權限來自廠商端。

### 3.2 資料模型（建議）

#### 3.2.1 核心表：廠商素材（實作時需建立）

廠商素材庫**必須有專用資料表**，實作時於資料庫新增。建議表名：`vendor_assets` 或 `manufacturer_materials`，欄位概念如下：

| 欄位 | 類型 | 說明 |
|------|------|------|
| `id` | UUID / bigint | 主鍵 |
| `manufacturer_id` | FK → manufacturers | 所屬廠商 |
| `category_key` | string | 訂製分類主分類 key（與現有 custom_product_categories 對齊） |
| `subcategory_key` | string | 子分類 key（可選） |
| `title` | string | 素材標題（例：iPhone 17 Pro Max 背蓋造型） |
| `description` | text | 簡短說明（可選） |
| `image_url` | string | 素材圖片 URL（Supabase Storage 或 CDN） |
| `usage_type` | string | 用途：如 `reference_only`（僅參考）、`quotable`（可依此報價）等，可擴充 |
| `is_public` | boolean | 是否開放給圖庫／搜尋（可選，預設 true） |
| `sort_order` | int | 排序（可選） |
| `created_at` / `updated_at` | timestamp | 建立／更新時間 |

- **分類**：與現有訂製分類（custom_product_categories / ai_categories）對齊，方便設計端依「分類」篩選廠商素材。
- **與「廠商作品」的差異**：廠商作品（portfolio）偏「成品展示」；廠商素材偏「可當設計參考的造型／結構／材質圖」，可與作品表分開，或於同一表用 `type` 區分（如 `portfolio` vs `material`），依現有 DB 架構擇一。

#### 3.2.2 權限與可見範圍

| 規則 | 說明 |
|------|------|
| 上傳與管理 | 僅該廠商（登入為該廠商帳號）可上傳、編輯、刪除自己的素材。 |
| 設計端使用 | 已登入設計者，在「產品設計」流程中可依分類／廠商篩選並選用素材作為參考圖。 |
| 對外曝光 | 若 `is_public` 為 true，可選擇是否在「圖庫找廠商」或搜尋中曝光（進階功能可後補）。 |
| 不下載原檔 | 設計端僅取得「用於設計的圖片 URL」（可為縮圖或固定解析度），不對外提供原檔下載，降低侵權風險。 |

### 3.3 廠商端：上傳與管理（必做）

**廠商必須具備素材上傳介面**：實作廠商素材庫時，除資料表與設計端選圖外，**廠商端一定要提供「素材上傳／管理」頁面或區塊**，讓廠商可上傳、編輯、刪除自家素材，設計端才會有內容可選。

#### 3.3.1 功能要點

- **入口**：廠商控制台（或「我的廠商作品」同層）新增「素材管理」或「可提供給設計者的素材」。
- **上傳介面**：提供**上傳表單**，可上傳多張圖片，每張需綁定：
  - 至少一個**分類**（主分類必選，子分類選填），與訂製分類一致。
  - 標題、簡短說明（選填）。
  - 用途（例：僅參考／可依此報價）。
- **列表與編輯**：列表顯示縮圖、標題、分類、用途、排序；支援編輯、刪除、調整排序。
- **上限與儲存**：每廠商可設素材數量上限（依營運／儲存成本訂），圖片存於 Supabase Storage（建議路徑如 `vendor-assets/{manufacturer_id}/{id}.jpg`）。

#### 3.3.2 介面建議

- **上傳介面**：與現有「廠商作品」上傳流程類似，但表單欄位改為：分類（必選）、子分類（選填）、標題、說明、用途、圖片；需有明確的「上傳」按鈕與成功／錯誤回饋。
- **管理介面**：可提供「依分類篩選」檢視自己的素材，列表支援編輯、刪除、排序，方便日後擴充時與設計端篩選邏輯一致。

### 3.4 設計端：廠商素材庫使用方式

#### 3.4.1 參考圖來源擴充

- **現有**：產品設計頁的參考圖來源包含「本機上傳」與「我的數位資產」。
- **新增**：增加第三種來源「**廠商素材庫**」。
- **操作**：在參考圖區塊提供切換或按鈕，例如「從數位資產選擇」旁增加「從廠商素材庫選擇」；點擊後開啟類似數位資產選擇的 modal，但資料來源改為廠商素材 API。
- **分類綁定**：素材庫載入的資料**以設計當下選取的分類為主**。開啟 modal 時，應帶入目前產品設計頁已選的「主分類」（及若有選「子分類」），作為 API 查詢的預設篩選條件，只顯示該分類下的廠商素材；設計者若尚未選分類，可提示先選分類再開素材庫，或由 API 回傳全部再由前端依選定分類再篩選（依產品設計頁流程擇一實作）。

#### 3.4.2 篩選與選圖

- **預設篩選（必做）**：素材列表**依設計當下選取的分類**載入——即與產品設計目前的主分類／子分類一致，不預設顯示全站所有分類的素材。
- **其他篩選維度**：
  - 在預設分類下，可再依**廠商**篩選（可選），方便「只想用 A 廠造型」的設計者。
- **呈現**：網格卡片（縮圖＋標題＋**廠商名＋分類**），點選即加入當前設計的參考圖列表；可多選，張數上限與現有參考圖上限一致。
- **帶入流程**：選中的素材圖 URL 帶入產品設計的參考圖欄位，與數位資產選圖相同，後續生成／分析流程不變。

#### 3.4.3 必須顯示「素材提供廠商」名稱與連結（必做）

雖然選用方式與數位資產庫一致，廠商素材庫**必須**讓設計者清楚知道素材來自哪一家廠商，以便設計完成後找該廠商下單：

| 情境 | 做法 |
|------|------|
| **選圖 modal 內** | 每張素材卡片除縮圖、標題、分類外，**必須顯示「提供廠商」名稱，且可點擊連至該廠商詳情頁**（如 `/vendor-profile.html?id=xxx` 或依現有廠商頁網址）。 |
| **參考圖列表／設計中** | 若當前參考圖包含來自廠商素材的項目，建議在該參考圖卡片或旁註標示「來源：OO 廠商」，並提供連結至該廠商頁，方便設計完成後前往詢價／下單。 |
| **生成後／找廠商** | 若本次設計曾使用廠商素材，可在「建議下一步」或找廠商流程中，優先提示「您使用了 XX 廠商的素材，可前往該廠商頁面詢價」。 |

實作時 API 回傳廠商素材列表需含：`manufacturer_id`、廠商名稱、廠商詳情頁 URL（或 slug），供前端顯示與連結。

#### 3.4.4 與數位資產的差異（設計端體驗）

- 數位資產：「我的數位資產」→ 只會看到自己帳號的圖。
- 廠商素材庫：「廠商素材庫」→ 看到多個廠商提供的素材，可依分類／廠商篩選；選用後同樣當作「一張參考圖」帶入，不改變既有生成／分析邏輯。

### 3.5 商業與授權（建議）

- 廠商上傳素材即表示**授權**本平台設計者在設計流程中使用該圖（作為參考圖、生成輸入等）。
- 若需更嚴謹，可在廠商上傳時增加「授權條款勾選」或「僅限本平台設計使用」之說明，並在設計端選用時簡短提示「此圖為廠商提供，僅供設計參考」。

### 3.6 實作順序建議

| 階段 | 內容 | 說明 |
|------|------|------|
| **1** | 定案資料模型與 API 契約 | 決定表名、欄位、與現有 manufacturers / 分類的關聯；設計端「取得廠商素材列表」API，**必傳設計當下選取之主分類（及子分類）** 作為主要篩選條件，可選傳廠商 ID。 |
| **2** | 廠商端：素材上傳與管理介面 | 廠商控制台新增「素材管理」：**上傳介面**（表單：分類、標題、說明、用途、圖片）、列表、編輯、刪除；圖片存 Storage；綁定分類。廠商端無上傳介面則設計端無素材可選，此階段為必做。 |
| **3** | 設計端：參考圖來源「廠商素材庫」 | 產品設計頁參考圖區塊新增「從廠商素材庫選擇」；modal 呼叫廠商素材 API、篩選（分類／廠商）、選圖帶入參考圖列表。 |
| **4** | 權限與可見範圍 | 實作「僅登入設計者可用」「廠商僅能管理自己的素材」；可選實作 `is_public` 與圖庫曝光。 |
| **5** | 進階（可選） | 圖庫找廠商頁面關聯「該廠有哪些素材」、搜尋時可篩選「有素材的廠商」等。 |
