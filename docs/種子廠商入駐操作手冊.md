# 種子廠商入駐操作手冊

**適用**：封測階段以「種子廠商模式」邀請真實廠商入駐；**由平台代為上傳作品，廠商不得編輯**。  
**規則**：種子廠商為 **90 天內公開展示**，不得自行編輯／上傳；若要編輯需至**挖貝付費**後升級。  
**還原點**：實作前狀態見 `docs/還原點-種子廠商實作前.md`。

---

## 管理員操作網址（請加入書籤）

| 環境 | 網址 |
|------|------|
| **本機** | **http://localhost:3000/admin/seed-vendors.html** |
| **正式站** | **https://你的網域/admin/seed-vendors.html**（例如 https://matchdo.cc/admin/seed-vendors.html） |

用**管理員帳號**登入本站後，開上述網址即可：**新增種子廠商**、查看列表、剩餘天數、是否已轉付費。頁面上有「複製網址」可複製當前網址。

---

## 後台操作步驟（代建立廠商 ＋ 代上傳素材）

全部在 **/admin/seed-vendors.html** 同一頁完成，不需複製 UUID、不需用 Postman。

### 步驟一：代為建立廠商資料

1. **綁定使用者**：下拉選單選一個「已註冊的帳號」（顯示 email／姓名）；該帳號登入後即代表此廠商。
2. **必填**：廠商名稱。
3. **選填**：地點、簡短介紹；勾選「顯示認證標章」；**聯繫與社群**（Email、電話、LINE ID、官網 URL、Facebook、Instagram）。
4. 按 **「建立」**。成功後該廠商會出現在下方列表。

### 步驟二：代為上傳素材（數位版型）

1. 在下方列表找到剛建立的廠商，該列有 **「上傳素材」** 按鈕，點下去會開 Modal。
2. 在 Modal 內填寫：
   - **主分類**：必選（與產品分類一致）。
   - **子分類**：選填。
   - **標題**、**介紹**、**造型**、**材質**：選填。
   - **素材圖**：選擇一張圖片檔案（必填）。
3. 按 **「送出」** 即代該廠商上傳一筆素材；可重複點「上傳素材」為同一廠商上傳多筆。

**說明**：上傳的是「素材」（數位版型／Base Models），供用戶在「再設計」時選為參考圖；不是廠商作品集的「作品」。廠商本人登入後可到廠商控制台「查看」素材，種子期間不得編輯。

## 如何調整種子廠商的角色（轉為付費）

種子廠商「角色」由 **`manufacturers.vendor_source`** 與 **`expires_at`** 決定：

| 狀態 | vendor_source | 效果 |
|------|----------------|------|
| **種子** | `'seed'` | 廠商不得編輯資料／素材；其素材僅管理員可見（避免未同意前之智慧財產權問題）。 |
| **付費／一般** | `null` 或非 `seed` | 廠商可自行編輯；其素材對所有使用者顯示。 |

### 方式一：後台一鍵「轉為付費」（建議）

1. 管理員登入 → 開啟 **/admin/seed-vendors.html**。
2. 在列表找到該種子廠商，該列會有 **「轉為付費」** 按鈕（僅尚未轉付費的廠商顯示）。
3. 點 **「轉為付費」** → 確認後送出。系統會將該廠商的 `vendor_source` 清空、`expires_at` 清空。
4. 轉換後：該廠商可自行編輯資料與素材，其素材也會在設計端「廠商素材庫」對一般使用者顯示。

### 方式二：API 調整

**PATCH /api/admin/manufacturers/:id**（Header：`Authorization: Bearer <管理員 token>`）

Body 可帶：

- **`vendor_source`**：設為 `null` 或 `"paid"` 即視為付費（非種子）。
- **`expires_at`**：設為 `null` 可取消／延長到期。

例：轉為付費廠商  
`PATCH /api/admin/manufacturers/<廠商 UUID>`  
Body：`{ "vendor_source": null, "expires_at": null }`

### 方式三：Supabase 手動改

在 Supabase → Table Editor → **manufacturers**，找到該筆廠商：

- 將 **vendor_source** 清空或改為 `paid`（或非 `seed` 的值）。
- 將 **expires_at** 清空（若不再依 90 天到期）。

## 種子廠商的資料該用什麼帳號編輯／上傳？

一律使用 **管理員帳號**（在 `profiles` 表裡 `role = 'admin'` 的那個登入帳號）：

| 操作 | 方式 |
|------|------|
| **建立種子廠商** | 管理員登入 → 開啟 **/admin/seed-vendors.html**，在「新增種子廠商」表單**選擇使用者**（下拉選單，顯示 email／姓名）、填寫廠商名稱後按「建立」。不需複製 UUID。 |
| **編輯廠商資料**（名稱、描述、地點、聯絡方式、分類、logo、expires_at、**vendor_source**） | 管理員登入 → 呼叫 **PATCH /api/admin/manufacturers/:id**（Body 帶要改的欄位）。或於 **/admin/seed-vendors.html** 對該列點 **「轉為付費」** 即可將種子改為付費廠商。 |
| **上傳／編輯／刪除作品** | 管理員登入 → 呼叫 **POST /api/manufacturers/:id/portfolio**、**PUT /api/manufacturers/:id/portfolio/:portfolioId**、**DELETE /api/manufacturers/:id/portfolio/:portfolioId**（`:id` 為該廠商的 manufacturer UUID）。 |

取得管理員 token：用管理員帳號在網站登入後，於瀏覽器開發者工具 → Application → Local Storage 查看 Supabase 的 session，或透過登入 API 取得 `access_token`，請求時 Header 帶 `Authorization: Bearer <token>`。  
**管理後台**：登入管理員帳號後，至 **後台 → 種子廠商管理**（/admin/seed-vendors.html）可：**新增種子廠商**（選使用者＋廠商名稱）、查看列表與剩餘天數、**代為上傳素材**（數位版型，每列「上傳素材」按鈕，選主分類＋素材圖即可）。不需複製 UUID、不需用 Postman。若需代為上傳「作品」圖請用 API POST /api/manufacturers/:id/portfolio。編輯廠商基本資料可至 Supabase 或呼叫 PATCH /api/admin/manufacturers/:id。

**重要：名稱與網址一律是「該廠商」的，不會顯示管理員帳號。**  
建立或編輯種子廠商時，請在 API Body 或 Supabase 中填寫**該廠商**的名稱、官網（`contact_json.url`）、聯絡方式等；前台廠商頁、作品列表顯示的名稱與網址都來自 `manufacturers` 表的那筆廠商資料，與「由誰登入上傳」（管理員或廠商本人）無關，**絕不會顯示管理員的帳號或聯絡方式**。

## 接下來要做什麼（您協助上傳、不讓廠商上傳）

1. **執行 SQL**（若尚未執行）：`docs/add-manufacturers-expires-at.sql`（新增 `expires_at` 欄位）。
2. **建立種子廠商**：管理員登入 → 開啟 **/admin/seed-vendors.html**，在「新增種子廠商」表單填寫綁定使用者 UUID、廠商名稱後按「建立」；或依 **方式二／方式三** 用 SQL／API 建立。系統會自動設 `expires_at = 現在 + 90 天`。
3. **由您上傳作品**：以**管理員帳號**登入後，使用 **POST /api/manufacturers/:id/portfolio**（multipart：`image`、`image_before` 等），URL 的 `:id` 填該種子廠商的 manufacturer UUID；後端會辨識管理員身份並允許上傳。廠商本人呼叫同一 API 會被拒絕（403）。
4. **廠商端**：種子廠商登入後可「看」控制台與作品，但**無法編輯廠商資料、無法新增／編輯／刪除作品與素材、無法建立／編輯／刪除資料夾（collections）**；若嘗試會得到「種子廠商由平台代為維護…請至挖貝升級付費方案」。前端會顯示藍色提示橫幅，並隱藏「新增作品／新增素材」、作品／素材的編輯與刪除按鈕。
5. **90 天後**：超過 `expires_at` 的廠商不會出現在首頁／搜尋／廠商列表；單一廠商頁會回 404。付費後可將 `expires_at` 清空或延長，並視需求將 `vendor_source` 改為非 seed 以開放編輯。

## 方式一：廠商已有本站帳號（僅當您允許廠商自建時用）

1. 請廠商登入後至 **廠商控制台**（或「我的功能」→ 開店／接單 → 建立廠商資料），填寫**廠商名稱**等必填欄位，建立一筆廠商資料。
2. 廠商本人即可在 **我的廠商作品** 上傳 3～5 組作品（對照圖／系列圖）；或由營運提供圖檔，由廠商登入後上傳。
3. （選用）在 Supabase 後台將該筆 `manufacturers` 的 `vendor_source` 設為 `'seed'`，並可設 `verified = true` 標示認證。

## 方式二：平台代建（廠商尚未註冊或由我們代建檔）

### 2.1 先取得廠商的登入帳號

- **A. 廠商自己註冊**：請廠商至本站註冊（email 或第三方登入），完成後提供您其**註冊用 email**，再到 Supabase → Authentication → Users 查詢該使用者的 **UUID (id)**。
- **B. 平台代建 auth 帳號**：在 Supabase → Authentication → Users 新增一筆 User（或透過 Auth API），設定 email／密碼，取得該使用者的 **id (UUID)**。請將登入方式告知廠商。

### 2.2 在資料庫建立廠商並綁定 user_id

在 Supabase SQL Editor 執行（將下列變數替換為實際值）：

```sql
-- 變數（請替換）
-- :user_uuid   = 步驟 2.1 取得的 auth.users.id (UUID)
-- :name        = 廠商名稱（必填）
-- :description = 簡短介紹（可選）
-- :location    = 縣市或地區（可選）

INSERT INTO public.manufacturers (
    user_id,
    name,
    description,
    location,
    contact_json,
    categories,
    is_active,
    verified,
    vendor_source
) VALUES (
    ':user_uuid',           -- 綁定登入帳號
    ':name',
    ':description',
    ':location',
    '{}'::jsonb,            -- 可改為 {"email":"","phone":"",...}
    '{}'::text[],           -- 可改為 ARRAY['服飾','家具'] 等，需與分類 key 一致
    true,
    true,                   -- 可設為 true 顯示認證標章
    'seed'
)
RETURNING id, name, user_id;
```

記下回傳的 `id`（廠商 UUID），用於後續上傳作品或貼上廠商頁連結。

### 2.3 上傳作品（3～5 組）

- **選項 A**：請廠商登入後到 **我的廠商作品** 自行上傳。
- **選項 B**：由營運以**管理員帳號**登入，用 **POST /api/manufacturers/:id/portfolio**（Header：`Authorization: Bearer <管理員 token>`，URL 的 `:id` 為該種子廠商的 manufacturer UUID）上傳；或到 Supabase 後台用 SQL 操作 `manufacturer_portfolio`。

### 2.4 提供廠商連結

廠商詳情頁：`https://matchdo.cc/vendor-profile.html?id=<廠商 UUID>`  
廠商控制台：請廠商登入後從首頁「我的功能」→ 廠商控制台進入。

## 方式三：使用後台 API 建立種子廠商

**POST /api/admin/seed-manufacturer**（僅管理員，Header：`Authorization: Bearer <管理員 token>`）

1. 取得廠商的 auth **user id**（見方式二 2.1）。
2. 以**管理員帳號**登入，取得 Bearer token。
3. 請求 Body（JSON）：
   - `user_id`（必填）：欲綁定之 auth 使用者 UUID
   - `name`（必填）：廠商名稱
   - `description`（選填）：簡短介紹
   - `location`（選填）：縣市或地區
   - `contact_json`（選填）：物件，如 `{ "email": "", "phone": "", "line_id": "", "url": "" }`
   - `categories`（選填）：陣列，如 `["服飾","家具"]`（需與站內分類 key 一致）
   - `verified`（選填）：true/false，是否顯示認證標章
4. 成功回傳 201 及 `{ id, name, user_id, vendor_source: "seed" }`；該廠商可登入後至「廠商控制台」上傳作品。

**注意**：需先於 Supabase 執行 `docs/add-manufacturers-vendor-source.sql` 新增 `vendor_source` 欄位，否則 API 會回傳錯誤提示。

---

## 注意事項

- 每位使用者（user_id）僅能綁定一筆廠商；若該 user 已建過廠商，不可重複建立。
- 種子廠商與一般廠商在**曝光與權限**上目前一致（皆依 `is_active`）；`vendor_source` 僅供標記與日後統計或擴充。
- 若日後改為「白手套認領」或分級付費，再參考 `docs/封測白手套入駐與權限-評估.md` 擴充。
